# 健身饮食记录（fitness-log-demo）

## 1. 项目名称
健身饮食记录（fitness-log-demo）

## 2. 项目简介
这是一个纯前端的示例项目，用来记录每日的健身和饮食相关数据。页面提供一个表单用于录入数据，并在下方展示历史记录和平均值，适合初学者了解：
- 基础 HTML 表单结构
- 使用 JavaScript 处理表单提交与数据校验
- 使用 `localStorage` 做本地持久化

> 说明：本项目不依赖后端服务，数据仅保存在你自己的浏览器中。

## 3. 当前功能
当前仓库已实现的功能如下：

- 新增一条记录，字段包括：
  - 日期
  - 体重（kg）
  - 腰围（cm）
  - 热量（kcal）
  - 蛋白质（g）
- 表单校验：
  - 必填项不能为空
  - 数值字段必须是大于等于 0 的数字
- 保存记录后即时提示“记录已保存”
- 历史记录列表展示（最新记录在前）
- 根据每条记录给出简单状态判断：
  - 蛋白质偏低
  - 热量偏高
  - 比较理想
  - 继续观察
- 自动计算并展示当前所有记录的平均值
- 打开页面时自动将日期默认填为当天

## 4. 使用方式
1. 在“新增记录”区域填写当天数据。
2. 点击“保存记录”。
3. 在“历史记录”区域查看：
   - 每条历史数据
   - 状态判断
   - 所有记录的平均值汇总

## 5. 本地运行方式
本项目是静态网页，可以直接在本地打开：

### 方式 A：直接打开
1. 下载或克隆仓库到本地。
2. 双击 `index.html`，用浏览器打开。

### 方式 B：使用本地静态服务（推荐）
如果你本机有 Python，可在项目目录运行：

```bash
python3 -m http.server 8000
```

然后在浏览器访问：

```text
http://localhost:8000
```

## 6. 数据存储说明（localStorage）
- 本项目使用浏览器 `localStorage` 保存数据。
- 存储键名为：`fitness_records_v1`。
- 数据格式为 JSON 数组，每一项是一条记录。
- 数据不会上传到服务器。
- 当你清空浏览器站点数据、切换浏览器，或使用无痕模式时，记录可能丢失或不可见。

## 7. 手动测试清单
下面是一份适合初学者的手动测试清单：

1. **空数据初始状态**
   - 首次打开页面时，历史记录显示“暂无历史记录”。
   - 平均值显示“暂无数据”。

2. **正常保存流程**
   - 输入一组合法数据并保存。
   - 预期：出现“记录已保存”提示，历史区新增一条记录。

3. **必填校验**
   - 留空任意字段直接提交。
   - 预期：出现错误提示，且不新增记录。

4. **非负数校验**
   - 在数值字段输入负数再提交。
   - 预期：出现错误提示，且不新增记录。

5. **状态判断检查**
   - 输入蛋白质小于 120 的数据，预期状态为“蛋白质偏低”。
   - 输入热量大于 2400 的数据，预期状态为“热量偏高”。
   - 输入热量在 2000~2400 且蛋白质不少于 120，预期状态为“比较理想”。

6. **平均值计算**
   - 连续保存多条记录。
   - 预期：平均值会随记录数量变化并正确更新。

7. **本地持久化**
   - 保存几条数据后刷新页面。
   - 预期：历史记录仍存在（同一浏览器同一站点下）。
